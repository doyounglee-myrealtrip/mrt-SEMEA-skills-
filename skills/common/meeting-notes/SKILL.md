# 회의록 자동 정리 스킬

Caret 녹취록 → 요약 → Confluence 작성 → Slack DM 순서로 실행한다.

## 1단계: 녹취록 요약

녹취록 전문을 분석해 아래 구조로 요약한다.

### 요약 구조
- **회의 제목**: 내용에서 추론 (없으면 날짜 + 주제)
- **일시**: 녹취록의 날짜/시간 정보 활용
- **참석자**: 발화자 이름 목록
- **회의 목적**: 1-2문장
- **주요 논의 내용**: 주제별로 구분해 핵심만 bullet로
- **결정 사항**: 회의 중 확정된 내용만
- **액션 아이템**: 담당자와 기한이 있는 할 일 목록

### 액션 아이템 형식
```
| 할 일 | 담당자 | 기한 |
|-------|--------|------|
| ...   | ...    | ...  |
```
기한이 언급되지 않은 경우 "미정"으로 표기한다.

---

## 2단계: Confluence 페이지 생성

요약이 완료되면 `scripts/post_confluence.py`를 실행한다.

```bash
python3 scripts/post_confluence.py \
  --title "<회의 제목>" \
  --content "<Confluence 위키 마크업으로 변환된 요약>"
```

### Confluence 페이지 형식 (위키 마크업)

```
h1. <회의 제목>

*일시:* <날짜 시간>
*참석자:* <이름1>, <이름2>

----

h2. 회의 목적
<목적>

h2. 주요 논의 내용
<bullet 리스트>

h2. 결정 사항
<bullet 리스트>

h2. 액션 아이템
|| 할 일 || 담당자 || 기한 ||
| ... | ... | ... |
```

---

## 3단계: Slack DM 전송

Confluence 페이지가 생성되면 Slack MCP를 사용해 나(doyoung)에게 DM을 보낸다.

### Slack 메시지 형식

```
📋 *회의록이 작성되었습니다*

*<회의 제목>*
<날짜 시간>

*핵심 내용*
• ...
• ...

*액션 아이템*
• [ ] <할 일> — <담당자> (<기한>)

🔗 <Confluence 페이지 링크>
```

---

## 에러 처리

- API 호출 실패 시 오류 내용을 사용자에게 명확히 전달한다
- Confluence 생성 실패 시 Slack 전송도 중단하고 원인을 안내한다
- 녹취록에 참석자/기한 정보가 불명확한 경우 추론하되 "(추정)"을 표기한다
